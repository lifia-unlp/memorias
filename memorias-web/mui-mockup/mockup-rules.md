Create an HTML+CSS mockup following Material Design principles.

Goals:
- Produce a static mockup that approximates the future Material UI design.
- Do not use Tailwind.
- Do not use inline styles.
- Do not style elements one by one with ad-hoc rules.
- Use semantic HTML and centralized CSS.

Styling rules:
1. Define design tokens in CSS variables:
   - colors
   - typography
   - spacing
   - border radius
   - shadows
   - breakpoints
2. Use reusable CSS classes for common patterns:
   - layout containers
   - cards
   - buttons
   - forms
   - navigation
   - lists
   - empty/loading/error states
3. Keep component classes generic and reusable.
4. Avoid page-specific styling unless necessary.
5. Follow Material Design conventions for spacing, elevation, typography, surfaces, and interaction states.
6. Do not use icons. 

Architecture:
1. Keep HTML structure semantic.
2. Keep all styling in one or more CSS files.
3. Do not use Tailwind classes.
4. Do not use Bootstrap or another UI framework.
5. Make the mockup responsive.

Important:
- This is only a mockup.
- It should be easy to translate later into MUI components and a centralized MUI theme.
